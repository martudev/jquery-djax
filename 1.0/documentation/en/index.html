This is the documentation of 'DJAX.js', is written in jquery and aims to simplify, dynamically, the loading of html content

{ DJAX v1.0 }

Installation

Not yet available

Introduction

First the DJAX must be started

var djax = new DJAX();
ó 
var djax = new DJAX({
      timeout: 2000, // default 5 seconds
      cache: false, // default browser cache of jquery ajax call disable
});

How to use it

We can use it in several ways, the first two fields of djax.djax () are jquery selectors, the first field accepts these formats "a", "form", "url: my_url", "['any_jquery_selector', 'any_attr' ] ".
The second field accepts this format "any_jquery_selector" or "undefined".
If in the first field we write "a", we will be obtaining the attribute 'href' of the element "a" as url. If we write "form", we will be getting the attribute 'action' of the element "form"

<div class="new-content">undefined</div>
<a class="test-load-url" href="/testLoad1"></a>
<script>
     djax.djax('a', '.new-content');
</script>
 
<script>
     djax.djax('url:/tesLoad1', '.new-content');
</script>

The second field also accepts these formats "['any_jquery_selector', 'before or replace or after']"

<div class="new-content">undefined</div>
<a class="test-load-url" href="/testLoad1"></a>
<script>
     djax.djax('a', ['.new-content', 'before']);
</script>

If we put "before", as in the example shown above, the result would be something like that. Keep in mind that if @MY_HTML has 'text' content, the result will not be the same as if @MY_HTML has html code content. This is because the browser shows the html content

Headers

By default the djax call has as header 'X-DJAX', but you can add more headers in the following way

<div class="new-content">undefined</div>
<a class="test-load-url" href="/testLoad1"></a>
<script>
     var headers = {'MY_HEADER': true, 'X-PJAX': true};
     djax.djax('a', '.new-content', headers);
</script>

Events listener

You can also define the so-called events listeners in the following way

<div class="new-content">undefined</div>
<a class="test-load-url" href="/testLoad1"></a>
<script>
     djax.djax('a', '.new-content', undefined, {
         success: function (data, stringData, jqXHR) {
             console.log(data);
         }
     });
</script>

By default all the events listeners are defined, for example if an error occurs in the call and the status is a 500, the following is shown

alert('Ups! Something went wrong');

The latter can be turned off as follows

<script>
     djax.djax('a', '.new-content', undefined, {
         options: {
             error: false, // this desactivates de default code of error
             success: false, // this desactivates de default code of success
         },
         success: function (data, stringData, jqXHR) {
             console.log(data);
         }
     });
</script>

At the moment the only available listener events are: "success", "error", "beforeSend"

.back()

This method allows you to go back to the previously modified html content, let's see an example

<div class="new-content">undefined</div>
<a class="test-load-url" href="/testLoad1"></a>
<script>
var djaxModel = undefined;
 
$('.update').click(function () {
     djaxModel = djax.djax('a', '.new-content', undefined, {
         success: function (data, stringData, jqXHR) {
             console.log(data);
         }
     }, {
        elementDom: '.new-content' // gets the html() from '.new-content' (required)
     });
});
 
$('.go-back').click(function () {
    djaxModel.then(function (model) {
        model.back();
    });
});
</script>

ó

<script>
var djaxModel = undefined;
 
$('.update').click(async function () {
     djaxModel = await djax.djax('a', '.new-content', undefined, {
         success: function (data, stringData, jqXHR) {
             console.log(data);
         }
     }, {
        elementDom: '.new-content' // gets the html() from '.new-content' (required)
     });
});
 
$('.go-back').click(function () {
    djaxModel.back();
});
</script>

Also if we want the new content obtained by the djax call after clicking on '.update' to be saved in the cache we should write the following

<script>
var djaxModel = undefined;
 
$('.update').click(async function () {
     djaxModel = await djax.djax('a', '.new-content', undefined, {
         success: function (data, stringData, jqXHR) {
             console.log(data);
         }
     }, {
        elementDom: '.new-content',
        cache: true
     });
});
 
$('.go-back').click(function () {
    djaxModel.back();
});
</script>

When writing the above mentioned, will cache the new content obtained by the djax call, so that if we press '.go-back' and then '.update', do not have to do a djax call again and therefore I would save transfer data

.cache()

This method allows to return the html content obtained from the djax call stored in the session, that is, when the user presses F5 the cached content will be erased

Let's see an example of how we make the geteo of the html content

<script>
 
$('.update').click(function () {
     djax.djax('a', '.new-content', undefined, {
         success: function (data, stringData, jqXHR) {
             console.log(data);
         }
     }, {
        name: 'cache1'
     });
});
</script>

Okay, now let's see an example of how to get the cache's html

<script>
var djax = new DJAX();
 
$('.update').click(function () {
     djax.djax('a', '.new-content', undefined, {
         success: function (data, stringData, jqXHR) {
             console.log(data);
         }
     }, {
        name: 'cache1'
     });
});
 
$('.get-cache').click(function () {
    var html = djax.cache('cache1');
});
</script>

Remember that with the previous code, the .back () function will continue to work normally
